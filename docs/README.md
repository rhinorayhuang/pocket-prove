## 文件用途

`architecture.md`
定義目前程式分層、責任邊界、資料流和擴充方向。

`development-rules.md`
定義必須遵守的實作規範、提交前檢查與禁止事項。

`folder-rules.md`
定義每個資料夾能放什麼、不能放什麼。

`writing-conventions.md`
定義命名、註解、文案、任務 ID、地圖 ID、資料表慣例。

`handoff.md`
提供下一個 chat session 或下一位開發者的接手方式。

## 文件優先順序

規範衝突時，優先順序如下：

1. `development-rules.md`
2. `architecture.md`
3. `folder-rules.md`
4. `writing-conventions.md`
5. `game-blueprint.md`

`game-blueprint.md` 主要是內容藍圖，不應推翻技術規範。
